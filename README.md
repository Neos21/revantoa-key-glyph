# KeyGlyph

<https://key-glyph.revantoa.workers.dev>


## コンセプト

- DevTools で DOM 構造や通信内容を見ても中身が容易には読み取れない、暗号化・難読化が施されたサイトを作る
    - あくまでネタ・実験的なモノであり、プロダクションレベルのセキュリティやパフォーマンス (実行速度) は求めない
    - HTTPS であれば「通信の傍受」自体は防げるモノであるが、「仮に」通信内容やソースコードを傍受したとしても、実データの復元が極めて困難な仕組みを探求する
- 画面内の好きな位置にテキストを書き込めるサイトを作る。他のユーザが書き込んだテキストも初期表示される
    - 「yourworldoftext」というサイトの UI に近いモノを想定している
        - GitHub ソースコード : <https://github.com/reverie/yourworldoftext>
        - デモサイト : <https://www.yourworldoftext.com/forexample>

### テキストの表示仕様

内部仕様は後述の「暗号化 (難読化) 仕様」に準拠した上で、エンドユーザ視点での UI 挙動をまとめる。

- テキストは「半角 : 1」「全角 : 2」の、一般的な「等幅フォント」で描画されることとし、「ピクセル単位」の座標値ではなく「行列」で位置を決める
- 画面上の任意の場所をクリックすることで、その位置に文字列を入力し始められる
- 同じ行列位置に上書きもできることとする
- 「全角 : 2」の列幅を取ることから、「あ」と書かれた部分の後半 1 列に「A」と入力した場合は、「(スペース)A」と入力されたモノとして上書きする
    - つまり、半角文字と全角文字が混在しても「行列が重なって見える文字」が生まれないようにする

### 暗号化 (難読化) 仕様

- バックエンドからフロントエンドにかけて、「他のユーザが書き込んだテキストと位置」の情報は平文でやり取りしない
    - 複数の API エンドポイントから、「位置情報」と「テキストの内容」を別々に取得し、フロントエンドでマージする
    - 「位置情報」および「テキストの内容」についても平文ではなく暗号化した情報を通信し、フロントエンドで復号する
    - 暗号化されたデータは、同一データであってもリクエストの度に異なる見た目になるようにランダム化するが、フロントエンドでは正確に復号できること
- フロントエンド上で、「他のユーザが書き込んだテキストと位置」の情報は DOM 要素内に直接表示されない
    - テキストとして人間は認識できるが、ソースコードを覗いてもコピペできる状態にならないように Canvas で描画する
- フロントエンドからバックエンドに「自分が書き込むテキストと位置」を送信する際も、平文でやり取りしない
    - 複数の API エンドポイントに、「位置情報」と「テキストの内容」を別々に送信し、バックエンドで単一の情報として処理する
    - フロントエンドで予め暗号化しておき、バックエンドで適宜復号する
    - 暗号化するデータは、同一データであってもリクエストの度に異なる見た目になるようにランダム化するが、バックエンドではその内容が正確に確認できること
- データベースに永続化する際は、Cloudflare Workers 上でも使える Web Crypto API を利用して暗号化した内容を保存する
    - データベースに永続化する内容は、同一のデータを「ランダム化」する必要はない。AES-GCM 等による通常の暗号化・復号で良い

- Cloudflare Workers : 実行基盤

- Hono : バックエンド API
- React + React Router v7 : フロントエンド
